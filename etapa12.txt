Quiero generar los módulos backend base en NestJS para el MVP PRO de un SaaS odontológico premium orientado a diseño de sonrisa y cierre comercial.

Objetivo:
crear una estructura backend limpia, modular, compilable y lista para crecer, con controladores, servicios, DTOs y validaciones básicas.

Stack:
- NestJS
- TypeScript
- PostgreSQL
- Prisma opcional pero recomendado

Reglas obligatorias:
- nombres de módulos, clases, métodos, DTOs, rutas y archivos en español
- no meter lógica compleja todavía
- primero construir CRUD limpio y mantenible
- usar class-validator en DTOs
- mantener separación clara entre controller, service y acceso a datos
- no sobrediseñar

Módulos mínimos a generar:
- autenticacion
- usuarios
- pacientes
- casos
- fotos_clinicas
- disenos
- presupuestos
- seguimientos
- notas_clinicas
- tareas

Necesito:
1. estructura de carpetas por módulo
2. controllers base
3. services base
4. DTOs crear/actualizar
5. validaciones básicas
6. integración inicial con Prisma si aplica
7. build compilable
8. tests base si corresponde

Entrega:
- backend modular y consistente
- código claro y mantenible
- preparado para conectar luego con frontend y lógica real

Al finalizar:
ejecutar npx prisma format, npx prisma validate, npm run build y npm run test si aplica
reportar errores encontrados, correcciones aplicadas y decisiones tomadas

Quiero integrar el módulo real de presupuesto y seguimiento comercial en el MVP PRO de un SaaS odontológico premium.

Objetivo:
reemplazar el mockup del módulo de presupuestos por datos reales, manteniendo la UI premium aprobada y una arquitectura limpia.

Necesito integrar:
- PresupuestoDetallePage
- crear presupuesto
- editar presupuesto
- opciones de tratamiento
- opción recomendada
- seguimiento comercial
- próxima acción
- timeline de seguimientos

Quiero:
- servicioPresupuestos
- servicioOpcionesTratamiento
- servicioSeguimientos
- hooks como usePresupuestoDetalle, useCrearPresupuesto, useActualizarPresupuesto, useOpcionesTratamiento y useSeguimientosComerciales
- manejo de loading, vacío, error y guardado
- componentes visuales desacoplados de la lógica de fetch
- tipos claros y consistentes en español

Restricciones:
- React + TypeScript + Tailwind en frontend
- NestJS en backend
- nombres en español
- no mezclar mocks con datos reales
- no romper la UI premium existente
- no poner fetch dentro de componentes presentacionales
- integrar de forma progresiva y mantenible
- no meter lógica fiscal o financiera compleja fuera del alcance del MVP

Entrega:
- detalle de presupuesto conectado a API real
- opciones de tratamiento funcionales
- seguimiento comercial funcional
- formularios de crear y editar funcionando
- arquitectura limpia con servicios y hooks
- eliminación de mocks que ya no se usen en este módulo

Al finalizar:
ejecutar npm run build, npm run lint y npm run test en frontend
ejecutar npm run build y npm run test en backend
reportar errores encontrados, correcciones aplicadas y pendientes reales

Quiero convertir el Editor de Sonrisa del MVP PRO en una herramienta funcional real, manteniendo la UI premium ya aprobada.

Objetivo:
pasar del mockup visual a un editor MVP usable que permita cargar la foto principal del caso, ajustar manualmente un diseño, comparar before/after y guardar borradores.

Necesito implementar:
- carga de paciente y caso
- carga de foto principal real
- carga de diseño existente si existe
- lienzo funcional
- overlay visual editable
- controles para ancho, alto, posicion_x, posicion_y, intensidad, forma y preset_aplicado
- comparador antes/después
- guardar borrador
- recuperar diseño existente
- si es razonable, duplicar versión

Quiero:
- useEditorSonrisa
- servicioDisenos
- servicioFotosClinicas
- componentes presentacionales desacoplados
- estado local editable claro y en español
- arquitectura preparada para futura evolución con visión por computadora

Restricciones:
- React + TypeScript + Tailwind en frontend
- NestJS en backend
- nombres en español
- no implementar IA avanzada todavía
- no mezclar lógica de API dentro de LienzoSonrisa ni PanelControlesSonrisa
- priorizar estabilidad, claridad y persistencia real
- si Canvas 2D alcanza, preferirlo sobre complejidad innecesaria

Entrega:
- EditorSonrisaPage funcional
- carga real de foto y diseño
- controles conectados al lienzo
- guardado y recuperación funcionando
- comparación before/after usable
- UI premium intacta

Al finalizar:
ejecutar npm run build, npm run lint y npm run test en frontend
ejecutar npm run build y npm run test en backend
reportar errores encontrados, correcciones aplicadas, decisiones técnicas tomadas y pendientes reales
